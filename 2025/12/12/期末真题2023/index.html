试题解析与参考答案

一、论述题解析

1. 传递基本类型和传递对象的区别

  • 解析： Java 中只有“值传递” (Pass by Value)。
  • 答案：
    • 基本类型： 传递的是数据的副本。在方法内修改变量的值，不会影响方法外的原始变量。
    • 对象（引用类型）： 传递的是对象引用（地址）的副本。虽然引用本身是拷贝的，但它指向堆内存中同一个对象。因此，在方法内通过引用修改对象的属性，会影响原来的对象；但如果在方法内将引用指向一个新的对象（obj = new Class()），则不会影响原来的对象。

3. ArrayList 和 LinkedList 的区别及其运用场景

  • 答案：
    • ArrayList： 基于动态数组实现。
      • 优点： 查询快（随机访问，时间复杂度 O(1)）。
      • 缺点： 增删慢（需要移动大量元素）。
      • 场景： 读多写少，需要频繁通过索引访问元素的场景。
    • LinkedList： 基于双向链表实现。
      • 优点： 增删快（只需修改指针，时间复杂度 O(1)）。
      • 缺点： 查询慢（需要从头遍历）。
      • 场景： 写多读少，需要在列表首尾频繁插入/删除的场景。

4. final 修饰属性、方法、类代表什么意思

  • 答案：
    • 修饰属性（变量）： 变为常量，初始化后不能被重新赋值（如果是引用类型，引用地址不可变，但对象内容可变）。
    • 修饰方法： 该方法不能被子类重写（Override），但可以被重载。
    • 修饰类： 该类不能被继承（例如 String 类）。

5. 线程安全问题 (Stack 类)

  • 解析： push 和 pop 方法操作共享变量 index 和 data 数组。

  • 情况： 会出现竞态条件 (Race Condition)。

    • 例如线程 A 和 B 同时执行 push。A 执行 data[index] = c 后，还没来得及 index++，线程 B 切入，也执行 data[index] = c，导致 A 的数据被覆盖。或者两个线程同时读取旧的 index，导致索引更新错误。

  • 修改： 给方法加上 synchronized 关键字。

    Java

6. 重载 (Overload) 与 重写 (Override)

  • 答案：
    • 方法 1 和 方法 2 (同一类中)： 如果方法名相同，参数列表不同，则是重载。
    • 方法 1 和 方法 3 (父子类中)： 如果方法名、参数列表、返回类型都相同（且访问权限符合规则），则是重写。如果不相同，则只是子类新增的方法，无特殊关系。

7. 接口 vs 抽象类 (“is-a” 关系)

  • 答案： 应该使用 抽象类 (Abstract Class)。
  • 理由：
    • 概念： “is-a” (是一个) 关系通常对应继承（Inheritance），即抽象类。接口通常代表 “can-do” (能做) 的行为契约。
    • 属性： 题目提到有“相似的属性”。抽象类可以定义普通成员变量（字段）来保存状态；接口只能定义 public static final 常量，不适合保存对象状态。
    • 方法： 题目提到“相似的方法”。抽象类可以提供普通方法的具体实现（代码复用）；接口在 Java 8 之前只能有抽象方法（Java 8+ 虽有 default 方法，但核心目的仍是解耦）。
    • 实现步骤： 使用 extends 关键字继承抽象类，可以直接复用父类的通用逻辑，只需重写差异部分。

8. String 和 StringBuilder 的区别

  • 答案：
    • String： 不可变字符序列。每次修改（如拼接）都会在内存中创建新的对象，效率低，浪费内存。适用于字符串内容不经常改变的场景。
    • StringBuilder： 可变字符序列。在原对象上进行修改，不产生垃圾对象，效率高。适用于需要频繁拼接、修改字符串的场景。
    • (补充：还有一个 StringBuffer，是线程安全的 StringBuilder)

9. 订单排序集合选择

  • 答案： 选择 TreeMap 或 ArrayList + Collections.sort。
  • 理由：
    • 如果需要集合在插入时就自动保持排序状态，使用 TreeMap，将订单编号作为 Key（因为 TreeMap 按照 Key 的自然顺序排序）。
    • 如果只需最后排序一次，可以使用 ArrayList 存储订单对象，然后让订单类实现 Comparable 接口（或使用 Comparator），调用 Collections.sort()。

10. 异常处理

  • 报错原因： method() 抛出了 MyException，这是一个受检异常 (Checked Exception)。Java 强制要求调用者必须处理受检异常。
  • 修改方法：
    1. 捕获处理： 使用 try-catch 块包裹调用代码。
    2. 继续抛出： 在 main 方法声明后添加 throws MyException。

11. super() 和 super.i

  • 答案：
    • super()： 调用父类的构造方法。必须放在子类构造方法的第一行。
    • super.i： 访问父类的成员变量 i。通常用于当子类有同名变量隐藏了父类变量时，强制访问父类成员。

import java.io.;
import java.net.;

// 主类：负责在门口接待客户
public class AreaServer {
public static void main(String[] args) {
// 题目未指定端口，这里假设用 8888
try (ServerSocket serverSocket = new ServerSocket(8888)) {
System.out.println(“服务端启动，等待连接…”);

        while (true) {
            // 1. 阻塞等待：直到有人连接才继续
            Socket socket = serverSocket.accept();
            
            // 2. 核心区别：一旦有人来，立马创建一个新线程去服务，主线程回去继续等下一个人
            new Thread(new AreaHandler(socket)).start();
        }
    } catch (IOException e) {
        e.printStackTrace();
    }
}

}

// 线程类：负责具体的计算任务
class AreaHandler implements Runnable {
private Socket socket;

public AreaHandler(Socket socket) {
    this.socket = socket;
}

@Override
public void run() {
    try {
        // --- 1. 准备流 ---
        InputStreamReader isr = new InputStreamReader(socket.getInputStream());
        BufferedReader in = new BufferedReader(isr);
        PrintWriter out = new PrintWriter(socket.getOutputStream(), true);

        String msg;
        // --- 2. 循环接收 ---
        while ((msg = in.readLine()) != null) {
            
            // 退出条件
            if (msg.equals("Bye")) {
                break; 
            }

            try {
                // --- 3. 业务逻辑：计算面积 ---
                // 将字符串转换为 double 类型 (例如 "5" -> 5.0)
                double side = Double.parseDouble(msg);
                double area = side * side;

                // 返回结果给客户端
                out.println("正方形面积为: " + area);

                // --- 4. 写入文件 (注意文件锁) ---
                // 题目要求：将客户输入的边长写入 D 盘 ca.txt
                // 使用 synchronized 锁住，防止多个人同时写文件导致冲突
                synchronized (AreaServer.class) {
                    // 技巧：在锁内部打开文件，写完立刻自动关闭 (try-with-resources)
                    try (PrintWriter file = new PrintWriter(new FileWriter("D:\\ca.txt", true), true)) {
                        file.println("客户输入边长: " + side);
                    }
                }

            } catch (NumberFormatException e) {
                // 如果客户输入的不是数字（比如 "abc"），要提示错误，防止服务器崩溃
                out.println("输入错误，请输入数字！");
            }
        }

        // --- 5. 收尾 ---
        socket.close();
        
    } catch (IOException e) {
        e.printStackTrace();
    }
}

}

(注：客户端代码虽然题目通常只要求写服务端，但为了完整性，逻辑是连接端口，发送读取的键盘输入，接收服务端的面积返回值)
